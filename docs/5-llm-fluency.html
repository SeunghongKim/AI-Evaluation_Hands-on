📝 모델 유창성 평가

DistilGPT-2가 생성한 텍스트의 자연스러움을 Perplexity와 규칙 기반 점수를 결합한 LLM-as-a-Judge 방식으로 측정합니다.

1. 유창성이란?

유창성(Fluency)은 생성된 텍스트가 문법적으로 올바르고, 자연스러우며, 일관성 있는지를 측정하는 지표입니다. LLM 품질 평가의 핵심 기준 중 하나입니다.

1
Very Poor
심각한 오류, 거의 이해 불가
2
Poor
여러 오류, 이해하기 어려움
3
Fair
명백한 오류, 이해는 가능
4
Good
사소한 문제, 전반적으로 양호
5
Excellent
완벽한 문법, 매우 자연스러움

2. 평가 방법 1: Perplexity

Perplexity(PPL)는 언어 모델이 텍스트를 얼마나 "예상 가능"하게 느끼는지를 측정합니다. 낮을수록 더 자연스럽고 유창한 텍스트입니다.

PPL(W) = exp(−1/N Σ log P(wi|w1...wi-1))
= exp(Cross-Entropy Loss)  ·  낮은 PPL → 모델이 예측하기 쉬운 텍스트 → 유창함
주의: 창의적이고 독특한 표현도 PPL이 높게 나올 수 있어 단독 지표로는 한계가 있음
Excellent
PPL < 50
→ Score 5
Good
50 ~ 100
→ Score 4
Fair
100 ~ 200
→ Score 3
Poor
200 ~ 400
→ Score 2
Very Poor
PPL > 400
→ Score 1

Perplexity 계산 코드

def calculate_perplexity(text, model, tokenizer):
    """GPT-2로 Perplexity 계산"""
    encodings = tokenizer(text, return_tensors="pt").to(device)

    with torch.no_grad():
        outputs = model(**encodings, labels=encodings["input_ids"])
        loss = outputs.loss

    perplexity = torch.exp(loss).item()
    return perplexity

3. 평가 방법 2: 규칙 기반 점수

Perplexity를 보완하는 규칙 기반 평가로 반복, 문장 완성도, 단어 다양성을 측정합니다.

평가 항목기준감점
문장 길이단어 수 < 5 → 너무 짧음-1.5점
반복 단어 비율고유어 비율 < 70% (반복 > 30%)-1.0점
연속 중복 단어"the the", "very very" 등 3회 이상-1.0점
문장 완성도마침표·느낌표·물음표로 끝나지 않음-0.5점
평균 단어 길이평균 3자 미만 (단음절 단어 집중)-0.5점
 
def rule_based_fluency_score(text):
    score = 5.0
    words = text.split()

    # 1. 반복 단어 비율
    repetition_ratio = 1 - (len(set(words)) / len(words))
    if repetition_ratio > 0.3:
        score -= 1.0

    # 2. 연속 중복
    consecutive = sum(1 for i in range(len(words)-1) if words[i] == words[i+1])
    if consecutive > 2:
        score -= 1.0

    # 3. 문장 완성도
    if not text.strip().endswith(('.', '!', '?')):
        score -= 0.5

    return max(1.0, min(5.0, score))

5. 두 평가 방법 비교

📐 규칙 기반

사전 정의된 규칙(반복, 길이 등)으로 점수를 계산합니다.

✅ 빠름, 해석 가능, 오프라인
❌ 맥락 이해 불가, 새로운 오류 패턴 탐지 어려움

🤖 Perplexity 기반

언어 모델이 텍스트의 확률을 계산하여 유창성을 측정합니다.

✅ 맥락 고려, 미묘한 표현 평가 가능
❌ 창의적 표현 과소평가, 계산 비용 높음

6. 개선 방향

생성 파라미터 튜닝

  • Temperature 0.6~0.7: 더 안전하고 유창한 출력
  • Repetition Penalty 1.2: 반복 단어 억제
  • no_repeat_ngram_size=3: 3-gram 반복 금지
outputs = model.generate(
    inputs,
    temperature=0.7,
    top_p=0.9,
    repetition_penalty=1.2,
    no_repeat_ngram_size=3,
    max_length=100
)

고급 평가 방법

  • BERTScore: 참조 텍스트와의 의미적 유사도 측정
  • GPT-4 as a Judge: 강력한 모델을 평가자로 활용
  • Human Evaluation: 사람이 직접 유창성 평가 (Gold Standard)

7. 유창성과 다른 품질 지표의 균형

트레이드오프설명
유창성 vs 창의성너무 안전한 출력은 지루하고 뻔할 수 있음
유창성 vs 사실성문법적으로 완벽해도 내용이 틀릴 수 있음
유창성 vs 다양성모든 출력이 비슷한 패턴이면 활용성 저하

다면적 평가 권장

유창성 단독으로는 LLM 품질을 충분히 평가할 수 없습니다. 유창성 + 관련성 + 사실성 + 다양성 + 안전성을 함께 측정하는 종합 평가 체계를 구축해야 합니다.
