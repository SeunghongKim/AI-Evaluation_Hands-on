🎯 모델 공정성 평가

FPR·FDR·TPR Parity로 Random Forest 분류기의 예측이 성별 그룹에 공정한지 측정하고, 임계값 조정으로 공정성을 개선합니다.

1. 공정성 평가 지표

모델 공정성은 보호변수 그룹 간 예측 성능의 차이를 측정합니다. 세 가지 핵심 지표를 사용합니다.

🚨

FPR Parity
False Positive Rate

FP / (FP + TN)

실제 음성 중 모델이 잘못 양성으로 예측한 비율. "억울한 피해"가 특정 그룹에 집중되지 않아야 합니다.

기준: 그룹 간 Gap < 0.10
✅

TPR Parity
True Positive Rate (Recall)

TP / (TP + FN)

실제 양성 중 모델이 올바르게 양성으로 예측한 비율. "기회(혜택)"가 모든 그룹에 공평하게 제공되어야 합니다.

기준: 그룹 간 Gap < 0.10
🎯

FDR Parity
False Discovery Rate

FP / (FP + TP)

양성으로 예측한 샘플 중 실제로는 음성인 비율. 예측 신뢰도가 그룹 간 균등해야 합니다.

기준: 그룹 간 Gap < 0.10

5. 임계값 조정으로 공정성 개선

# 다양한 임계값에서 공정성 변화 측정
y_pred_proba = model.predict_proba(X_test_scaled)[:, 1]

thresholds = np.linspace(0.1, 0.9, 9)
results = []

for thresh in thresholds:
    y_pred = (y_pred_proba >= thresh).astype(int)
    mf = MetricFrame(
        metrics={'fpr': false_positive_rate, 'tpr': true_positive_rate},
        y_true=y_test, y_pred=y_pred, sensitive_features=sensitive_test
    )
    results.append({
        'threshold': thresh,
        'fpr_gap': abs(mf.by_group.loc['Male','fpr'] - mf.by_group.loc['Female','fpr']),
        'tpr_gap': abs(mf.by_group.loc['Male','tpr'] - mf.by_group.loc['Female','tpr'])
    })

6. 공정성 개선 전략

알고리즘 레벨

  • Fairness Constraints: 학습 시 공정성 제약 조건 추가
  • Adversarial Debiasing: 보호변수를 예측하지 못하도록 적대적 학습
  • 가중치 조정: 그룹별 손실 함수 가중치 변경

후처리 레벨 (ThresholdOptimizer)

  • 그룹별로 다른 분류 임계값을 적용하여 FPR·TPR Gap 최소화
from fairlearn.postprocessing import ThresholdOptimizer

postprocessor = ThresholdOptimizer(
    estimator=model,
    constraints="equalized_odds"
)
postprocessor.fit(X_train, y_train, sensitive_features=sensitive_train)
y_pred_fair = postprocessor.predict(X_test, sensitive_features=sensitive_test)

⚡ 공정성 vs 정확도 트레이드오프

공정성 제약을 강하게 적용할수록 전체 모델 정확도가 약간 감소할 수 있습니다. FPR Parity ↔ TPR Parity 동시 만족은 수학적으로 불가능(불가능성 정리)한 경우가 있으므로, 도메인별로 중요한 지표를 선택해야 합니다.
